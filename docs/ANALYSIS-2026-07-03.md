## 一、架构现状

```
market.db: stocks(5525), daily(16.7M), benchmark_daily(1573), factor_registry(16行/6active)
trades.db: sim_trades(动态), strategy_config(种子本金)

config.yaml → 结构参数 (佣金/滑点/因子阈值)
strategy_config.initial_capital → 种子本金 (backtest.py 启动写一次)
sim_trades.capital_after → 运行时资产 (每笔交易自动更新, 唯一真相源)

factor/ → factor_registry 表驱动, 6 active 因子
pipeline.py → 7步串联
web/ (Flask, 端口8521)
```

## 二、设计原则

1. **因子审计入DB**: 因子增删 = UPDATE factor_registry.status + 写 status_reason。不再手改 Python dict。
2. **资本单一真相源**: config 不存运行时状态。资本链 = sim_trades.capital_after → strategy_config.initial_capital。
3. **静态配置 vs 运行时状态**: config.yaml 只放佣金率/滑点/因子阈值等结构参数。资本/持仓/交易记录都在 DB。
