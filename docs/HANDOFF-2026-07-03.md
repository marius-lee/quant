## 全盘代码分析

### 架构总览

系统采用 **7 层 Pipeline** 架构:

```
Layer 0: data/     — SQLite 数据仓库 (store.py 962行 god-class)
Layer 1: config/   — YAML 配置热加载
Layer 2: factor/   — 11动态因子 + 5基本面因子
Layer 3: alpha/    — 空壳 (已合并到 pipeline + synth)
Layer 4: risk/     — 约束过滤 + 行业/市值中性化
Layer 5: optimizer/— 组合构建 (资本自适应三档)
Layer 6: execution/— 模拟执行 + 成本模型
Layer 7: monitor/  — 绩效报告 + Brinson 归因
Web:    web/       — Flask 仪表盘 (端口 8521)
```

数据流: `DataStore → Factor → Alpha合成 → Risk过滤 → Optimizer → Execution → Monitor → Web`
