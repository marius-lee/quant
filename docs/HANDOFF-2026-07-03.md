## 系统概述

A 股量化选股系统, 7 层 Pipeline 架构:
```
Step 1 数据 → Step 2 加载 → Step 3 因子 → Step 4 风险 → Step 5 优化 → Step 6 执行 → Step 7 监控
```

- **本金**: ¥5,000 (strategy_config 表, 不是 config.yaml)
- **基准**: 沪深 300 (000300)
- **调仓**: 每 2 周 (10 个交易日)
- **仓位**: 全仓 1-3 只
- **数据源**: tencent (qfq) + akshare (qfq) + pytdx (手算 qfq)
- **数据库**: market.db (日线/股票/基本面/因子) + trades.db (交易记录)

## 数据库表结构

### market.db
- `stocks`: symbol, name, market(SH/SZ/BJ), industry, pe, pb, total_mv, roe, eps, bvps, ...
- `daily`: symbol, date, open, high, low, close, volume, amount, turnover (PK: symbol+date)
- `benchmark_daily`: index_code, date, open, high, low, close
- `factor_registry`: name, status(active/deprecated/experimental), ic_mean, ic_ir, status_reason
- `lhb_detail`: symbol, trade_date, close, ... (空表)
- `northbound_flow`: date, symbol, net_buy, ... (空表)

### trades.db
- `sim_trades`: date, symbol, side, price, shares, pnl, pnl_pct, capital_after, strategy
- `strategy_config`: strategy, initial_capital

## 决策日志 (按时间倒序, 用于历史回溯)

每条决策必须包含: 日期、决定内容、原因、替代方案（被否决的选项）。

### 2026-07-03 13:00 — 全项目 Tab→Space 格式化
- **决定**: 43 个 .py 文件全部 tab→4 spaces + 去行尾空白
- **原因**: 126 commits 中 ~10 个是缩进/语法错误（formatting 混用 tab/space 导致 sed/perl 脚本写出残破代码）。每次手改都引入新的格式错误。
- **替代方案**: 逐文件修 → 否决（会无限循环）；引入 black/autopep8 → 否决（不想加依赖）
- **教训**: 永远在项目初期统一格式化，不要等 126 commits 后再做

### 2026-07-03 12:56 — 添加 validate.py 自动检查
- **决定**: 每次 backtest 前自动跑 4 项检查（config vs code / factor一致性 / DB结构 / 参数）
- **原因**: 126 commits 中 ~40 个是 bug fix，大部分属于 "改 A 忘改 B" 模式。自动检查能拦下最常见的问题。
- **替代方案**: 写单元测试 → 否决（量化策略测试成本高，一个回测 5 分钟）

### 2026-07-03 12:50 — turnover limit 从默认 0.5 改为 0.0
- **决定**: 之前"移除" turnover limit 只改了 config.yaml 注释，函数默认值还是 0.50
- **原因**: 单仓切换换手率 >50% 时被 silently scale down，导致分批卖出、持仓碎片化
- **教训**: 改配置必须搜所有引用（函数签名 + 调用方），不能只改一个地方

### 2026-07-03 12:43 — 回测 -23.6%, 集中仓位 max_positions 3→1
- **决定**: ¥5,000 全仓 1 只而非分散 3 只
- **原因**: 交易记录显示同只股票反复买卖（300251 买了 4 次），分散仓位增加噪声
- **结果**: -23.6% → -11.4%

### 2026-07-03 12:40 — 退役 momentum_10d, reversal_5d, turnover_rev_5d
- **决定**: 只保留 4 个强因子 (|IC| > 0.02): bp_ratio, size, gap_5d, amihud_20d
- **原因**: 数据修复后重测 IC，3 个因子跌破阈值。Phase 3 的"多样性"论证基于脏数据，不成立
- **替代方案**: 保持 7 因子 → 否决（IC=0.008 的因子不提供多样性，只提供噪声）
- **详见**: FACTOR-AUDIT-2026-07-03.md Phase 4

### 2026-07-03 12:35 — 数据修复后 IC 重算, 发现 3 个因子方向反转
- **决定**: reversal_5d 和 momentum_10d 从 -cum 改为 +cum（反转→动量）
- **原因**: 基于 Sina 未复权脏数据测的 IC 完全错误。干净数据下 A 股短期是动量效应，不是反转
- **教训**: 数据源的微小差异（未复权 vs 复权）可以完全改变因子评估结论

### 2026-07-03 11:00 — 移除 Sina 数据源
- **决定**: 从 all_sources 删除 Sina，仅保留 tencent + akshare
- **原因**: Sina 返回未复权数据（验证：688167 除权日 -35.9%，穿越科创板 ±20% 涨跌停）
- **教训**: 加入新数据源之前必须验证复权类型，不能只看速度
