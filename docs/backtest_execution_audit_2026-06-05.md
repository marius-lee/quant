# 回测执行逻辑审计

日期: 2026-06-05

## 当前执行逻辑

`engine/backtest_runner.py` → `run_backtest()`

```
① predict_all() → 对所有股票做一次预测 → pred_series

② 只在测试期第一天选一次股 (entry_date = test_close.index[0])
   过滤: 买得起的 + 非涨停的 + 当日有数据的

③ 选 top N (默认3只)

④ _execute_positions():
   等权分配资金 → 100股取整 → 买入
   持有整个测试期 (不调仓, 不重选)
   每日按收盘价计算权益

⑤ 移动止损: 从峰值回撤 > max_drawdown(35%) → 立即平仓 → 冻结

⑥ 最终卖出扣佣金+印花税 → 算Sharpe
```

## 5 个致命缺陷

| # | 问题 | 后果 |
|:--:|------|------|
| 1 | 只选一次股 — 测试期第一天选了, 467天不换 | 第一天挑的股票可能半年后已经不行了 |
| 2 | 不调仓 — 持有整个测试期 | 不是"策略", 是 buy-and-hold 3只 |
| 3 | 35% 止损 — 牛市中正常回调20-30%触发平仓 | 刚跌完就反弹, 已经被清仓了 |
| 4 | 等权 100股取整 — ¥5000买3只, 资金碎片化 | 每只买一点点, 手续费比例高 |
| 5 | 一次预测覆盖全期 — 用今天的因子预测1年后的走势 | 因子信号有保质期, 最多5天有效 |

## 根因分析

```
ML 选股质量:   ✅ Top 10 平均 +9.79% 5日收益, 8/10 日期跑赢随机
最终回测收益:  ❌ Sharpe -0.30, ¥3788 (亏损)

差距来源:
  1. 止损过早触发 (牛市中正常回调就平仓)
  2. 手续费摩擦 (¥5000本金下, 每笔¥5佣金, 高频交易被吃)
  3. buy-and-hold 不重选 (top 3 持有整个测试期)
  4. 等权 100 股取整 (资金利用率低)
```
