# 关键 Bug 修复 — 2026-05-30

## Bug 1: 因子缓存 real_fundamental 因子键格式错误崩溃

**文件:** `factor/real_fundamental.py:29-41`
**严重程度:** 高（因子缓存构建时崩溃）

**根因:** 因子键使用单元素元组 `(f"real_ep_{sym}",)`，导致 `pd.MultiIndex.from_tuples()` 生成单层 MultiIndex，`stack(level=1)` 抛出 IndexError。

**修复:** 改为双元素元组 `("real_ep", sym)`，使 MultiIndex 有 2 层，`stack(level=1)` 将 stock 符号从列展开为行索引。

```python
# Before (broken)
factors[(f"real_ep_{sym}",)] = ...

# After (fixed)
factors[("real_ep", sym)] = ...
```

## Bug 2: 行业市值中性化静默失效

**文件:** `web/pipeline.py:44-45` → `engine/ranker.py:65`
**严重程度:** 高（中性化从未执行，推荐结果被行业和市值偏差污染）

**根因:** `apply_demon_and_neutralize(..., None)` 传入 `None` 作为 `stocks_repo`，被 `_neutralize` 内的裸 `except Exception` 静默吞掉。

**修复:** 传入 `StockRepo(self.store)` 替代 `None`。同时给 `_neutralize` 的异常捕获添加 `logger.warning(...)` 日志。

```python
# Before
pred_series = apply_demon_and_neutralize(
    pred_series, self.store, self.demon, None
)

# After
from data.repository import StockRepo
pred_series = apply_demon_and_neutralize(
    pred_series, self.store, self.demon, StockRepo(self.store)
)
```

## Bug 3: 中性化异常无日志

**文件:** `engine/ranker.py:65-66`
**严重程度:** 低（掩盖 bug #2）

**修复:** `except Exception: return scores` → `except Exception: logger.warning(...); return scores`
