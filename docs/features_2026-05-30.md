# 核心功能实现记录 — 2026-05-30

基于全面审查报告第二部分（缺失核心功能 2.2.1-2.2.7）逐项实现。

---

## 2.2.1 风控止损

**文件:** `backtest/event_engine.py`, `engine/backtest_runner.py`

- `EventDrivenBacktest.__init__` 新增 `max_drawdown`（最大回撤）和 `daily_loss_limit`（日亏损限制）参数
- 每日估值后检查：回撤超限 → `_liquidate()` 强平所有持仓；日亏损超限 → 强平
- `_liquidate()` 方法：清空待买入、卖出全部持仓、标记 trade reason
- `backtest_runner.py` 从 `config.yaml` `risk.*` 读取参数传入引擎

## 2.2.2 滚动重训练

**文件:** `engine/backtest_runner.py`

- `signal_fn` 闭包新增滚动重训练逻辑
- 每次进入测试期或距上次重训练 ≥ `retrain_freq`(21天) 时，用最近 `train_window`(504天) 数据重新训练模型
- 训练参数从 `config.yaml` `strategy.*` 读取
- 重训练失败时记录 warning，继续使用当前模型

## 2.2.3 基准对比

**文件:** `data/store.py`, `engine/backtest_runner.py`, `engine/builder.py`, `web/templates/index.html`

- `DataStore.get_benchmark()` — 从 tushare 拉取基准指数（沪深300）日线收益
- `backtest_runner.py` 回测后计算基准对比指标：基准年化收益、基准夏普、超额年化、超额夏普
- `builder.py` 将 benchmark 数据透传到结果 dict
- 前端回测面板新增基准对比区块

## 2.2.4 自动因子生成（WorldQuant 风格）

**文件:** `factor/alpha_factory.py` (新建), `factor/cache.py`

- 11 种基础算子随机组合：rank, delta, ts_mean, ts_std, ts_max, ts_min, scale, log, abs, momentum, vol_ratio
- `generate()` — 生成 N 个候选因子，IC 筛选保留 top N
- 输出 (date,stock) MultiIndex DataFrame，直接兼容 factors_cache 格式
- `factor/cache.py` 缓存更新时自动追加自动生成因子

## 2.2.5 通知告警

**文件:** `auto_run.py`, `web/templates/index.html`

- `_write_status()` 新增 `alerts` 参数，将新进/掉出/高分信号写入 `auto_status.json`
- 日志用 `***` 前缀醒目打印告警信息
- 前端读取 `auto_status.json` 在 header 下方显示粉色告警条

## 2.2.6 K 线图

**文件:** `web/app.py`, `web/templates/index.html`

- 新 API `/api/kline/<symbol>` — 返回最近 120 天 OHLCV 数据（直接 SQL 查询，轻量快速）
- 前端引入 ECharts CDN
- 新 Tab「K 线图」— 点击推荐行加载该股票的蜡烛图
- `loadKline(sym)` 函数：调用 API、初始化 ECharts、渲染蜡烛图

## 2.2.7 实时价格刷新

**文件:** `web/templates/index.html`

- 前端 `loadTrack()` 改为命名函数，`setInterval(loadTrack, 30000)` 每 30 秒自动刷新
- 往日回顾面板自动更新最新价格和涨跌幅
