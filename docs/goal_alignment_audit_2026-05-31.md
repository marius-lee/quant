# 北极星目标校准审计报告

**审计标准**: ¥5,000 初始资金, 6个月内 → ¥1,000,000 (200倍回报)
**审计日期**: 2026-05-31
**审计范围**: 全部 34 个 Python 文件 + config.yaml
**审计结论**: 当前系统是稳健量化选股系统，非极值收益捕获系统。**方向正确但参数完全反了。**

---

## 核心发现

四个独立审计维度（策略层、因子层、数据层、架构层）得出一致结论：

> 这套管线的每一层设计都在主动消灭极端收益的来源。它不是有问题——它是被设计成这样的。机构要的是夏普比率最大化，你要的是绝对收益最大化。这两个目标函数是正交的。

---

## 一、最致命的 5 个阻碍

### 1. 行业+市值中性化 — ranker.py `_neutralize()`

**杀伤力: 10/10**

这是对激进策略伤害最深的功能。它的工作机制：
- 对行业dummy和log市值做多元回归，取残差作为"纯化信号"
- 如果AI板块整体很强，所有AI股都会因为"属于AI行业"而被扣分
- 如果小盘股表现出超额收益，它们会因"市值小"被扣分

**对5000→100万: 行业集中和市值暴露是你主动想获取的alpha来源，不是要消除的噪音。** 妖股往往集中在热点行业，中性化会系统性地降低所有热点行业股票的得分。

### 2. 30只持仓 + 100万初始资金

**杀伤力: 10/10**

`config.yaml` 默认值 `max_positions: 30, initial_capital: 1_000_000`。

5000元买30只股票：每只167元。A股每手100股，最低股价约2元=200元/手。30只持仓意味着**无一只股票买得起一手**。

现实约束：5000元最多买 2-3 只股票（每只约1500-2500元）。

### 3. 股票池排除最高弹性标的

**杀伤力: 9/10**

`StockRepo.get_qualified()` 过滤了：
- **ST股票** — 但ST摘帽恰是A股弹性最大的事件之一（摘帽日经常涨停）
- **次新股（<250天）** — 注册制新股前5日无涨跌幅，次新是A股最活跃板块
- **北交所** — 30%涨跌幅，是10%主板的3倍

### 4. 训练目标错位

**杀伤力: 8/10**

- 训练目标：**未来5日收益率** — 模型学到的是"未来几天大概率微涨"，不是"明天涨停"
- 模型评价：**Rank IC** — 奖励"对所有股票排序正确"，不奖励"准确捕获极端收益的Top 5只"
- 集成方式：**5模型加权平均** — 集成的本质是降低方差，而200倍需要的正是识别尾部事件
- **Ridge线性模型** — L2正则化线性模型天然偏好均值回归，系统性地低估极端收益

对200倍目标，更需要：预测**明天是否涨停**（二值分类），或用**尾部IC**（只评估Top 5%收益股票的排序能力）替代全局IC。

### 5. 回测没有涨跌停约束

**杀伤力: 8/10**

生产管线用的向量化回测 (`backtest_runner.py`):
- 不检查涨停 → 推荐的涨停股在回测中"买入"，实盘买不到
- 不检查跌停 → 回测可以"卖出"跌停股，实盘卖不掉
- 不扣手续费（万三对5000元账户无意义→最低5元/笔=有效费率0.5%）
- 不做T+1、不做手数取整

用这个回测跑出的 Sharpe 1.24 对实盘无参考价值——它连交易费都没扣。

## 二、因子体系评估

### 高价值因子（保留并放大）

| 因子类别 | 贡献度 | 理由 |
|---------|--------|------|
| 博弈论因子 (28个) | **10/10** | Amihud=流动性补偿, PIN=知情交易, Kyleλ=价格冲击——直接捕获信息不对称，恰是极端收益来源 |
| 妖股信号 (demon.py) | **9/10** | 量能突变+价格突破+异常波动+连续阳线=妖股启动信号，与目标完全对齐 |
| 技术因子中高波动部分 | **7/10** | volatility_5d, turnover_chg_5d, volume_ratio_5d, ma_deviation_5d |

### 低价值/反向因子（应删除或限制）

| 因子类别 | 贡献度 | 理由 |
|---------|--------|------|
| 真实基本面 (real_fundamental.py) | **0/10** | real_log_mv引导模型选大市值/高价股→5000元买不起；PE/PB/股息对短期极端收益无意义，且是反向信号 |
| 代理基本面 dollar_volume | **2/10** | 成交金额代理=选大成交额股票，与"小盘爆发"相反 |
| 反转因子 (reversal) | **3/10** | 直接惩罚趋势，对追涨停策略是纯噪音 |
| 质量因子 (quality_proxy = -波动) | **2/10** | 奖励低波动，抑制高弹性标的 |
| 60日长周期因子 | **3/10** | 3个月前的信息对明天涨停几乎无关 |
| 自动生成因子 (alpha_factory) | **4/10** | IC阈值0.01过低→大量噪声因子；且固定目标为5日收益 |

## 三、缺失的关键能力

### P0 — 不解决则实盘无法运行

| 缺失能力 | 说明 |
|---------|------|
| 5000元资金约束 | 必须在选股阶段排除"买不起"的股票（股价×100 > 可用资金/top_n） |
| 涨跌停检测 | 回测必须标记当日涨停/跌停的股票为"不可交易" |
| 正确的佣金模型 | 5000元级别：万三费率是错的→每笔最低5元（有效费率0.5-1%），卖出加千一印花税 |
| 事件驱动回测启用 | 当前的向量化回测（backtest_runner.py）无手续费/滑点/T+1/涨跌停。必须切到 event_engine.py |

### P1 — 实盘跑起来会暴露

| 缺失能力 | 说明 |
|---------|------|
| 下单链路 | 从推荐列表→券商API→实际订单的全链路完全空白 |
| 可买性过滤 | StockRepo.get_qualified 没有价格、日成交额、板块权限过滤 |
| 小盘流动性风险 | 日成交额<500万的股票，5000元买入不是问题，卖出可能无人接盘 |
| 北交所支持 | 30%涨跌幅，但需要新数据源和股票列表 |

## 六、底线判断

**以5000→100万为标准，当前系统的目标对齐度: 3/10。**

核心问题不是代码质量——代码质量是好的。问题是**所有设计决策都指向保守**：分散持仓（降波动）、行业中性（消alpha）、价值因子（选蓝筹）、浅树模型（防过拟合）、长周期训练（求稳定）、集成模型（取平均）。

修复P0和P1改动后，目标对齐度可以提升到 **7/10**。剩余差距需要从可转债、龙虎榜数据、日内策略等方向弥补。

**最重要的一句话：这个系统最大的问题不是技术问题，是参数问题。改参数不改代码，就能让系统从"稳健"转为"激进"。**
