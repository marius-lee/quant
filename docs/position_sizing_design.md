# 仓位分配设计

日期: 2026-06-05

## 设计原则

信号强的重仓, 信号弱的轻仓。如果一只鹤立鸡群, 全押。只有信号相近时才分散。

## 算法

`strategy/position_manager.py` → `allocate_concentrated()`

```
score² 加权 → 信号差距被平方放大

top1 权重 > 60% → 全押一只
top1 权重 ≤ 60% → 按 score² 权重分散

手数取整为 100 股, 买不起的跳过
剩余零钱不买 (留给手续费缓冲)
```

## 实例

```
例 1: 鹤立鸡群
  000518 score=0.85 → 权重 92% → ¥4,600
  000700 score=0.42 → 权重 8%  → ¥400

例 2: 信号相近
  000518 score=0.60 → 权重 45% → ¥2,250
  000700 score=0.55 → 权重 38% → ¥1,900

例 3: top1 > 60% 全押
  000518 score=0.90 → 权重 100% → ¥5,000 全仓
```

## 来源

- 等权因子组合基线: DeMiguel et al. (2009)
- Kelly Criterion: 小资金不适用 (¥5000 下每笔都是满仓)
- 集中持仓: ¥5000 策略需要极高周转率 → 200 倍目标
