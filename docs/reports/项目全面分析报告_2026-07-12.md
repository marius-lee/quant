# 项目全面分析报告 — 2026-07-12

## 1. 项目是干什么的?

**A 股量化选股系统**, 基于 Grinold & Kahn 的 **Fundamental Law 框架**搭建。

**北极星目标**: ¥5,000 -> ¥100 万 (6 个月, 200 倍)。

**核心流程**:
- 每日盘后 15:30 触发调度器
- 7 层流水线: 数据 -> 因子(70个) -> Alpha 合成 -> 风控中性化 -> 组合优化 -> 模拟执行 -> 监控归因
- 信号自底向上流动, 订单自顶向下执行
- Web 仪表盘实时查看状态 (Flask, 端口 8521)

**关键约束**: 小资金起步(¥5,000), A 股 T+1, 整手交易(1手=100股), 涨跌停限制(±10%)。

### 量化选股框架 — 除了 Grinold & Kahn 还有哪些?

| 框架 | 核心思想 | 适用场景 |
|------|----------|----------|
| **Grinold & Kahn Fundamental Law** | IR = IC × sqrt(breadth) | 因子策略 — 当前项目采用 |
| **Black-Litterman** | 市场均衡 + 投资者观点 -> 后验收益 | 组合构建, 可与因子得分融合 |
| **Fama-French 多因子** | 市场/规模/价值/动量/盈利/投资 6 因子 | 基准归因, 风格暴露监控 |
| **Barra 风险模型** | 风险因子分解 + 协方差预测 | 专业级风控, MSCI 标准 |
| **Kelly Criterion** | 最优下注比例 = (胜率×赔率-败率)/赔率 | 仓位管理 |
| **风险平价 (Risk Parity)** | 等风险贡献 | 多资产配置 |
| **HMM 状态切换** | 牛市/熊市/震荡 三状态 | 自适应因子权重 |
| **强化学习 (RL)** | 状态->动作->奖励 | 端到端交易策略 |

**建议**: 当前 Grinold & Kahn 框架适合因子驱动的选股。可逐步融入:
1. **Black-Litterman** 融入组合构建层 (factor scores -> investor views -> posterior returns)
2. **Barra** 风格因子作为风险暴露监控维度
3. **Kelly** 作为仓位管理优化 (替换当前固定 max_positions)
4. **HMM** 市场状态检测 (已在 requirements.txt 中引入)

| **Kelly 仓位管理** | P2 | 用历史胜率/赔率算最优仓位, 替代固定 max_positions=20 |

### 因子有效性危机的根因分析

**现状**: 70 因子中 1 个 active (zt_streak), 2 个 candidate, 7 个 registered, 其余全部 rejected/retired。

**可能原因** (待逐一验证):

1. **IC 评估窗口遇上风格切换** — 2026 年 A 股风格快速轮动, 因子 IC 天然低
2. **因子注册门槛设置过高** — factor_registry 的 min_abs_ic=0.02 / min_icir=0.5 过滤掉了大量有微弱但稳定信号的因子
3. **前视偏差** — 回测中是否无意用了未来数据? 需逐因子检查
4. **实现偏差** — 因子公式是否与原始论文/业界标准一致? 需审计
5. **幸存偏差** — 评估用股票池是当前上市股还是历史全量? 后者才正确
6. **复权处理** — 分红送转是否前复权? 不复权会导致收益率计算错误
7. **zscore 过滤过度** — zscore_min_count_dense=30 可能过滤掉了不少小票信号
