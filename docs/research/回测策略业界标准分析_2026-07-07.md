# 回测策略业界标准分析 — 2026-07-07

## 当前流程

eval_stepwise.sh:
- 120 天全量数据 = IC 计算 + 因子选择 + 回测验证（三步同数据）
- 步进回测：逐个因子加入，IR 提升则保留
- 问题：全样本内（in-sample），无样本外验证，无多重检验修正

## 四阶段标准流程（提议）

```
数据时间线（按日期排序，从旧到新）
│←── 训练期 (60%) ──→│← 验证期 (20%) →│← 测试期 (20%) →│
│ IC计算+因子选择     │ walk-forward调参 │ 最终评估（只用一次）│
```

来源综合：Grinold & Kahn (1999) Active Portfolio Management + De Prado (2018) Advances in Financial Machine Learning + Harvey, Liu & Zhu (2016)

### 阶段 1: 训练期 IC 筛选
- Spearman rank IC t-test (t ≥ 2.0)，仅用训练期数据

### 阶段 2: 验证期 walk-forward
- Rolling window: 每 20 天滚一次，评估新因子是否提升 OOS IR

### 阶段 3: 测试期最终评估
- 选定因子集合，测试期跑一次（不做调参）

### 阶段 4: 统计稳健性
- DSR（Deflated Sharpe Ratio）修正多重测试偏差
- FDR（Benjamini-Hochberg）控制假阳性率

## 待验证项

60/20/20 比例和四阶段命名是综合多家文献后的提议，非单一来源直接引用。
业界通用标准的核心只有一条：**模型训练和性能评估必须用不同时间段的数据**（walk-forward out-of-sample）。
具体实现（split 比例、滚动窗口大小、重采样频率）各机构按资金规模和数据量调整，没有唯一标准。
